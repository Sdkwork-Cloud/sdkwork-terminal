# 并行车道交付包与集成交接标准

## 1. 每个并行车道必须交付

- 变更目录清单
- 变更摘要
- 测试命令与结果
- 风险点
- 是否阻塞其他车道

## 2. 推荐交付包结构

- `scope.md`：范围、owner、主写目录、共享契约
- `changes.md`：核心变更、影响面、兼容性说明
- `tests.md`：命令、结果、未覆盖项
- `risks.md`：已知风险、阻塞项、升级人
- `rollback.md`：回滚方式、回滚前提、不可逆项
- `contract-delta.md`：仅在协议、错误体、事件族变化时提交
- `docs-delta.md`：涉及 `docs/架构`、`docs/step` 时必须提交
- `blockers.md`：未解决阻塞、依赖车道、升级等级

## 3. 交接要求

- 只通过公共契约和包根接口交接
- 禁止要求其他车道依赖内部实现细节
- 必须说明是否修改协议、错误体、事件族
- 必须说明是否影响 `docs/架构`、`docs/step`、`release-manifest.json`
- 子 agent 交接时也必须遵循同一交付包结构

## 4. 集成前检查

- 是否越过包边界
- 是否引入冲突命名
- 是否遗漏测试
- 是否需要同步回写 `docs/架构` 或 `docs/step`
- 是否触发共享模型重新冻结
- 是否需要提升到波次级集成窗口处理

## 5. 升级阻塞条件

- 修改共享契约但未同步通知依赖车道
- 修改 Session 模型、ExecutionTarget、Release Manifest 但无单 owner 审批
- 缺失测试结果、回滚说明或风险说明
- 交付包未说明是否可直接收口
