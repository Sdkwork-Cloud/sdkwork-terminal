# 2026-04-10 v0.2.40 - Terminal Fidelity Smoke Harness

## Changed

- Added `tools/smoke/terminal-fidelity-probe.mjs` as a repeatable real-terminal smoke harness for `OSC title`, `alternate screen`, `bracketed paste`, `mouse-reporting`, and `CJK` rendering.
- Added `tools/smoke/terminal-fidelity-probe.ps1` as the Windows wrapper entrypoint aligned with the current workstation baseline.
- Added parser-backed regression coverage so the smoke harness contract is testable even when interactive terminal assertions still require manual execution inside a real PTY tab.
- Added `--report-template --platform <platform> --shell <shell>` so platform smoke runs can start from a structured archival template instead of free-form notes.
- Added `--review-template --platform <platform> --shell <shell>` so manual smoke review can start from a repository-owned markdown checklist instead of ad-hoc notes.
- Updated Step 05 architecture, step, prompt, and smoke docs so remaining TUI evidence is now required to flow through repository-owned smoke entrypoints.

## Verified

- `node --experimental-strip-types --test tests/terminal-fidelity-probe.test.ts`
- `node tools/smoke/terminal-fidelity-probe.mjs --print-plan`
- `node tools/smoke/terminal-fidelity-probe.mjs --sample-analysis`
- `node tools/smoke/terminal-fidelity-probe.mjs --report-template --platform ubuntu --shell bash`
- `node tools/smoke/terminal-fidelity-probe.mjs --review-template --platform windows --shell powershell`

## Remaining Gap

- This adds a professional smoke entrypoint, but it does not yet replace cross-platform human execution on `Windows / Ubuntu / macOS`.
- `IME` and full-screen TUI application evidence still need platform-by-platform runs and archived results.
