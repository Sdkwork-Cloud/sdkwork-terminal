# 2026-04-11 v0.2.40 Xterm Mutation Write Serialization

## Summary

The xterm viewport driver now serializes terminal mutations through one internal queue. Full refresh, reset, and raw runtime writes no longer race each other.

## Changes

- Added one internal mutation queue for `render(...)`, `reset()`, and `writeRaw(...)`.
- Added `writeTerminalContent(...)` so driver writes share one completion path.
- Recomputed render plans inside the queued mutation, so snapshot refresh decisions use the latest terminal state at execution time.
- Added source-contract coverage for serialized terminal mutation handling.

## Impact

- Prevents reset/write overlap during rapid PTY output.
- Reduces stale frame flashes, duplicate content, and cursor-order instability on the active tab.
- Improves `new tab`, tab reactivation, and runtime replay stability without changing the PTY contract.
