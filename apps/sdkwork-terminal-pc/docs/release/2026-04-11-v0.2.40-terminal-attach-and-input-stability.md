# 2026-04-11 v0.2.40 Terminal Attach and Input Stability

## Scope

- Real desktop PTY attach timing
- Hidden-tab xterm lazy mount
- First-attach input binding correctness
- Regression coverage for attach-only xterm open/bind behavior

## Changed

- `packages/sdkwork-terminal-shell/src/index.tsx`
  - `TerminalStage` now defers xterm attach until the tab first becomes active.
  - First attach now hydrates from live refs for runtime mode, PTY content, snapshot, title listener, and input listener instead of mount-time closures.
- `packages/sdkwork-terminal-infrastructure/src/index.ts`
  - `ensureRuntime()` no longer opens xterm implicitly when a container is present.
  - `attach(...)` remains the only path that performs `open + waitForNextAnimationFrame + fit + bindInputDisposables`.
  - Pre-attach `setInputListener(...)` and `setTitleListener(...)` now store listeners without eagerly creating/opening runtime state.
- `tests/shell-app-render.test.ts`, `tests/terminal-view-driver.test.ts`
  - Added regression coverage for lazy attach and attach-only open/bind sequencing.

## Fixed

- Fixed hidden tabs pre-opening xterm against invisible panels, which could surface stale geometry, cursor drift, and bad first-activation focus.
- Fixed a real driver ordering bug where listener setup could pre-create runtime state before attach and skip the intended first-frame fit/bind path.
- Fixed desktop tabs showing terminal chrome without consistently wiring live xterm input on the first real attach.

## Verified

- `node --experimental-strip-types --test tests/shell-app-render.test.ts tests/terminal-view-driver.test.ts tests/shell-runtime.test.ts tests/shell-tabs.test.ts`
