# 2026-04-10 v0.2.40 Terminal Bootstrap Recovery

## Summary

本轮继续停留在 `Wave B / Step 05`，只收口真实 terminal 启动恢复主链，不推进到 `Step 06`。目标是把 desktop shell 的启动失败、自动重试、手动重启与 queued input 保留做成专业终端级闭环。

## Changed

- `packages/sdkwork-terminal-shell/src/model.ts`
  - 新增显式 `retrying` runtime state。
  - 新增 `runtimeBootstrapAttempts / runtimeBootstrapLastError`，使启动恢复成为可测试状态语义。
  - 新增 bootstrap retry/reset helper，支持自动重试与手动重启差异化处理。
- `packages/sdkwork-terminal-shell/src/index.tsx`
  - Desktop shell 启动失败时自动重试一次，不做无限重试。
  - `binding / retrying` 期间继续输入时，原始输入统一进入 `runtimePendingInput`。
  - 自动重试耗尽后 overlay 明确提示 `Automatic retry was exhausted`。
  - `failed` 启动失败 tab 执行 `Restart shell` 时保留 queued input，避免恢复时丢按键。
