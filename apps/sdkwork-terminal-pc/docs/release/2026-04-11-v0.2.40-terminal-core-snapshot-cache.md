# 2026-04-11 v0.2.40 Terminal Core Snapshot Reference Cache

## Scope

- Terminal-core snapshot reference stability
- Repeated snapshot read cost reduction
- Downstream rerender pressure reduction
- `new tab` performance chain hardening

## Changed

- `packages/sdkwork-terminal-core/src/index.ts`
  - Added `WeakMap<TerminalCoreState, TerminalSnapshot>` caching in `getTerminalSnapshot(...)`.
  - Reused the same snapshot object for repeated reads against the same immutable terminal state object.
- `tests/terminal-core-workbench.test.ts`
  - Added regression coverage that unchanged terminal-core state reuses both the snapshot reference and the `visibleLines` reference.

## Fixed

- Fixed terminal-core recreating a fresh snapshot object on every read even when the state object had not changed.
- Reduced downstream React/xterm churn caused by unstable snapshot identity during repeated workbench polling and shell snapshot reads.
- Removed one more source of avoidable allocation from the `new tab` and tab-switch render path.
