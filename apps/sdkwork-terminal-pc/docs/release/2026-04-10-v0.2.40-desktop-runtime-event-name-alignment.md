# 2026-04-10 v0.2.40 Desktop Runtime Event Name Alignment

## Summary

This round fixes a desktop runtime hot-path incompatibility: the app subscribed to session stream events with dotted names like `sdkwork-terminal.runtime.v1.session.output`, but Tauri `listen(...)` rejects event names containing `.`. The session itself was healthy; the desktop event bus naming was not host-safe.

## Changed

- `packages/sdkwork-terminal-infrastructure/src/index.ts`
  - Desktop runtime session subscriptions now encode event names as Tauri-safe topics, replacing `.` with `:`.
- `src-tauri/src/lib.rs`
  - Desktop host runtime event emission now uses the same Tauri-safe topic encoding.
- `tests/desktop-runtime-bridge.test.ts`
  - Added a regression guard that asserts desktop subscription event names contain no `.` and match Tauri's allowed character set.
