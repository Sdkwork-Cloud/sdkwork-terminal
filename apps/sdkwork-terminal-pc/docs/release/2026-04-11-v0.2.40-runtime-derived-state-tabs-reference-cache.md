# 2026-04-11 v0.2.40 Runtime Derived State Tabs-Reference Cache

## Summary

`ShellApp` no longer rebuilds runtime derived-state structures on rerenders when `snapshot.tabs` is unchanged.

## Changes

- Added a `WeakMap` cache keyed by `snapshot.tabs` reference for `createRuntimeDerivedState(...)`.
- Reused cached derived state on reference hit instead of rebuilding the snapshot lookup map, candidate arrays, and effect keys.
- Added source-contract coverage for cache lookup, cache hit return, and cache population on miss.

## Impact

- Reduces repeated derived-state allocation during pure UI rerenders that do not change terminal shell state.
- Lowers one more source of `new tab` and high-output UI churn by keeping runtime derived structures reference-stable when the snapshot is stable.
- Keeps runtime bootstrap, retry, and pending-input effect behavior unchanged while tightening rerender cost.
