# 2026-04-19 v0.2.52

## Release Strategy

- This release advances the shipped `v0.2.51` line with focused hardening for desktop launch correctness, release/dev parity, Windows packaging behavior, and terminal input fidelity.
- The scope is intentionally narrow: no new product mode is introduced, and the release centers on making the existing desktop terminal implementation behave correctly and predictably in packaged Windows environments.

## Wave / Step

- Wave B
- Step 07 continued
- Desktop terminal launch and release hardening

## Changed

- Added `packages/sdkwork-terminal-shell/src/shell-app.css` and tightened the desktop HTML bootstrap contract so packaged desktop builds load the same xterm viewport and helper-textarea behavior as development builds.
- The desktop profile menu now carries explicit status descriptors for launcher failures and WSL discovery health instead of relying on console-only diagnostics.
- Local Tauri build/dev wrappers now materialize portable generated config overlays with explicit Node-backed `beforeBuildCommand` and `beforeDevCommand` wiring, and the desktop dev config keeps a dev-only identity split away from release packaging.

## Fixed

- Fixed large terminal paste so payloads are chunked instead of truncated, and Unicode surrogate pairs are preserved across chunk boundaries.
- Fixed the desktop working-directory picker bridge so it cannot block forever, and picker failures now reopen the profile menu with a visible launch error.
- Fixed WSL launcher refresh so failed discovery does not immediately discard the last successful WSL distribution list.
- Fixed packaged Windows desktop startup so release builds use the GUI subsystem rather than surfacing an extra console window.
- Fixed release/dev rendering drift by explicitly loading terminal shell styling in the packaged desktop host.
