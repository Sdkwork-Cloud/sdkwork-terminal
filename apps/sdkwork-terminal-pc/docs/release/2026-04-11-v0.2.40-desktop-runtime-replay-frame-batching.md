# 2026-04-11 v0.2.40 Desktop Runtime Replay Frame Batching

## Changed

- `packages/sdkwork-terminal-shell/src/index.tsx` now queues live desktop PTY replay by session and flushes it once per frame instead of mutating shell state for every incoming stream event.
- Batched replay flush now applies the coalesced entry list per session/tab and acknowledges desktop attachments with `batch.maxSequence`.
- `tests/shell-app-render.test.ts` now freezes the frame-batching and batch-ack contract.

## Fixed

- Reduced live-output scheduling pressure that could make `new tab`, tab switching, and direct interaction feel stalled while busy PTY sessions streamed output.
- Reduced unnecessary acknowledgement churn by collapsing desktop ack writes to the last sequence actually flushed from the queued replay batch.
- Removed the stale test expectation that still enforced the old per-entry ack path after the runtime replay implementation had moved to batched flushing.
