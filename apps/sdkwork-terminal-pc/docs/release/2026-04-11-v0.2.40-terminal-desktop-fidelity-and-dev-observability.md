# 2026-04-11 v0.2.40 Terminal Desktop Fidelity and Dev Observability

## Scope

- Desktop terminal tab switch fidelity
- PTY bootstrap cursor and overlay alignment
- Windows local shell interactive input/output verification
- Desktop dev and Tauri diagnostic transparency

## Changed

- `packages/sdkwork-terminal-shell/src/index.tsx`
  - Active desktop terminal tabs no longer replay the full raw PTY buffer with `writeRaw(..., true)` on every activation.
  - Xterm cursor visibility now follows the real bootstrap overlay state instead of rendering early during PTY bind/retry.
- `packages/sdkwork-terminal-shell/src/model.ts`
  - Real runtime tabs now start with an empty transcript surface; fallback seed lines are reserved for web local-shell fallback only.
- `packages/sdkwork-terminal-shell/src/index.tsx`
  - Fake prompt rendering and fake `runTerminalShellCommand(...)` fallback are now explicitly limited to `web + local-shell` tabs only.
- `packages/sdkwork-terminal-shell/src/model.ts`
  - Added `shouldUseTerminalShellFallbackMode(...)` as the single truth for fallback-tab eligibility, so seed transcript, fake prompt, and fake command execution all share one rule.
- `packages/sdkwork-terminal-shell/src/model.ts`, `packages/sdkwork-terminal-shell/src/index.tsx`, `tests/shell-app-render.test.ts`
  - Runtime vs fallback rendering truth is now locked to helper predicates end to end, eliminating the last source-test dependency on literal `remote-runtime` checks inside `TerminalStage`.
- `packages/sdkwork-terminal-shell/src/model.ts`, `packages/sdkwork-terminal-shell/src/index.tsx`, `tests/shell-runtime.test.ts`
  - Added `resolveTerminalStageBehavior(...)` so runtime stream, fallback prompt, and bootstrap overlay now come from one stage-level decision and stay regression-tested across desktop PTY, web fallback, and web remote-runtime paths.
- `packages/sdkwork-terminal-shell/src/model.ts`, `packages/sdkwork-terminal-shell/src/index.tsx`, `tests/shell-tabs.test.ts`
  - Added `resolveTerminalShellRuntimeClientKind(...)` so runtime client routing now keeps `remote-runtime` web-only and keeps desktop `local-shell / connector` tabs on the desktop bridge instead of relying on scattered component branches.
- `packages/sdkwork-terminal-shell/src/index.tsx`
  - Idle runtime bootstrap now surfaces a direct failure when a tab requires a runtime client but none is available, preventing tabs from sitting forever in `idle` with no operator signal.
  - Viewport paste handler registration now tracks `showLivePrompt` and `onViewportInput`, so a tab that transitions from fallback to real runtime no longer keeps a stale fallback paste path.
  - Active-tab focus recovery now tracks `showLivePrompt` and `usesRuntimeTerminalStream`, so a tab that changes interaction mode while already active re-focuses the correct real terminal surface.
  - Active-tab focus recovery now yields while terminal search is open, so search input focus stays stable even if the tab changes interaction mode underneath it.
  - TerminalStage measurement and initial attach now read `active` through a live ref instead of a mount-time closure, so tabs created in the background still measure, resize, and align correctly when first activated.
  - Active-tab focus recovery now also tracks `showBootstrapOverlay`, so focus and viewport sync are re-armed when the desktop bootstrap overlay disappears on the first PTY frame.
- `packages/sdkwork-terminal-infrastructure/src/index.ts`
  - Xterm driver now waits one animation frame before the initial `fitAddon.fit()` and before viewport measurement, reducing wrong cols/rows and top-left cursor drift when a terminal surface becomes visible after layout changes.
  - Viewport measurement now retries across a small number of animation frames before giving up, reducing stale PTY geometry when the terminal surface stabilizes slightly after activation.
- `packages/sdkwork-terminal-shell/src/index.tsx`
  - Terminal chrome CSS now forces the xterm helper textarea caret transparent, adding a second guard against native browser caret leakage over the real PTY cursor.
  - Active-tab recovery now waits for viewport measurement before restoring terminal focus, reducing activation races where focus could land before PTY geometry had been re-fit.
- `apps/desktop/src/main.tsx`, `apps/web/src/main.tsx`
  - Removed `React.StrictMode` around the terminal app entrypoints so PTY/session/bootstrap side effects are not intentionally double-mounted in development.
- `tests/shell-app-render.test.ts`
  - Added regression coverage that desktop/web entrypoints render `<App />` directly without `StrictMode`.
- `packages/sdkwork-terminal-infrastructure/src/index.ts`
  - Xterm theme initialization now applies cursor visibility at creation time, not only after runtime attachment.
- `crates/sdkwork-terminal-pty-runtime/tests/local_shell_session_runtime.rs`
  - Windows smoke now verifies real interactive command input and streamed output before termination.
- `tools/scripts/run-tauri-dev.mjs`
  - Dev startup now prints selected desktop URL and generated Tauri config path.
  - Cleanup now also runs on process exit and `SIGBREAK`.
  - Startup and shutdown now attempt to release stale workspace-scoped desktop Vite servers on the default dev port.
- `tools/scripts/run-vite-host.mjs`
  - Desktop Vite host startup now prints the exact command and working directory.
- `tools/scripts/run-tauri-cli.mjs`
  - Tauri CLI invocations now print the effective subcommand.
  - `tauri info` now strips unsupported `--config` flags and runs in `apps/desktop` context for accurate package diagnostics.
  - Module import no longer auto-executes `tauri help`; the CLI now runs only when invoked as a program.
- `apps/desktop/package.json`
  - `tauri:info` now uses `node ../../tools/scripts/run-tauri-cli.mjs info`.
- `tests/run-tauri-cli.test.mjs`, `tests/workspace-structure.test.mjs`, `tests/shell-app-render.test.ts`, `tests/terminal-view-driver.test.ts`
  - Added regression coverage for desktop cursor/bootstrap behavior, Tauri CLI normalization, and package script truth.

## Fixed

- Fixed desktop tab activation resetting xterm state, scroll position, and cursor placement.
- Fixed bootstrap-stage cursor leakage that could render a blinking cursor at the top-left before the first PTY frame.
- Fixed web `remote-runtime` tabs inheriting fallback seed transcript text before the first real runtime frame.
- Fixed real runtime tabs being able to drift back into fake command execution paths when no runtime session was yet attached.
- Fixed `apps/desktop` `tauri:info` failure caused by unsupported `--config`.
- Fixed Tauri info package detection so desktop diagnostics now report `@tauri-apps/api` correctly.
- Fixed script import side effects that were executing `tauri help` during tests.
- Fixed stale workspace Vite servers on `127.0.0.1:1420` not being reclaimed before the next desktop dev run.
- Fixed missing release-grade evidence for Windows local shell interactive command roundtrip.
- Fixed a latent runtime routing inconsistency where `remote-runtime` bootstrap could choose the web client while later input/client resolution still defaulted to desktop mode branches.
- Fixed silent idle bootstrap stalls for runtime tabs that required a client but had none available.
- Fixed stale viewport paste closures that could keep routing pasted text through the fallback prompt path after a tab switched into real runtime mode.
- Fixed active-tab focus recovery being tied only to tab activation, which could leave focus on the old input target when a tab changed interaction mode in place.
- Fixed terminal focus restoration stealing focus from the search input when an active tab changed interaction mode while search overlay was open.
- Fixed inactive-at-mount tabs keeping a stale `props.active=false` viewport-measure closure, which could leave newly activated tabs with wrong PTY geometry, cursor placement, and terminal interaction.
- Fixed bootstrap-overlay dismissal not re-triggering focus/measurement recovery, which could leave the live PTY surface visually ready but not fully re-armed after startup.
- Fixed xterm fit/measure running before the terminal surface had a stable post-activation layout, which could leave newly shown tabs with stale PTY size and cursor placement.
- Fixed a remaining browser-level caret leakage path where the helper textarea could still surface a native blinking caret instead of only the real terminal cursor.
- Fixed a remaining activation ordering race where terminal focus could be restored before the viewport re-measure finished, leaving newly shown tabs more likely to surface cursor/focus drift.
- Fixed a remaining hidden-to-visible activation edge where the first post-activation frame could still report unstable terminal geometry and leave the PTY on stale cols/rows.
- Fixed dev-only double mount of terminal/PTTY side effects from `React.StrictMode`, which could otherwise duplicate bootstrap, attach, and subscription work during local desktop iteration.
