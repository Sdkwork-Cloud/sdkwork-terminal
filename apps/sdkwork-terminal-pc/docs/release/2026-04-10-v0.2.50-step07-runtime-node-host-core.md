# 2026-04-10 v0.2.50 Step 07 Runtime-Node Host Core

## Summary

This round closes the missing pure Rust host core between the Step 07 web shell binding and the future HTTP/SSE wrapper. `remote-runtime / server-runtime-node` no longer stop at request modeling or UI wiring; the repository now owns a real runtime-node host that can create PTY-backed sessions, stream output, terminate cleanly, and recover persisted session truth from sqlite.

## Changed

- `crates/sdkwork-terminal-runtime-node/src/host.rs`
  - Added `RuntimeNodeHost`.
  - Added `RemoteRuntimeSessionCreateRequest`.
  - Added session index, replay, input, binary input, resize, terminate snapshots.
  - Added `RuntimeNodeStreamEvent`.
  - Closed the server-side loop `create -> attach -> PTY spawn -> replay/state -> input / input-bytes / resize / terminate -> output / warning / exit -> sqlite recovery`.
- `crates/sdkwork-terminal-runtime-node/src/lib.rs`
  - Re-exported the new host API from the crate root.
- `crates/sdkwork-terminal-pty-runtime/src/lib.rs`
  - Changed `terminate_session()` to remove the runtime handle before kill.
  - Treated Windows `os error 6 / 1460` as best-effort close so explicit-command PTY sessions can still drain `Exit`.
  - Added terminate closeout regression coverage.
- `crates/sdkwork-terminal-runtime-node/tests/runtime_node_host_test.rs`
  - Added real host lifecycle coverage for output and exit.
  - Added sqlite recovery rebuild coverage.

## Verification

- `cargo test --manifest-path crates/sdkwork-terminal-runtime-node/Cargo.toml -- --nocapture`
