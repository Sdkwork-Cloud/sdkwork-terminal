# 2026-04-10 v0.2.40 Terminal Binary Input

## Summary

This iteration stays in `Wave B / Step 05` and closes the remaining desktop input-fidelity gap around `xterm.onBinary`.

## Changed

- `packages/sdkwork-terminal-infrastructure/src/index.ts`
  - adds `TerminalViewportInput`
  - subscribes to both `xterm.onData` and `xterm.onBinary`
  - converts binary-string payloads into raw bytes with `binaryStringToBytes(...)`
  - adds desktop bridge method `writeLocalShellInputBytes(...)`
- `packages/sdkwork-terminal-shell/src/model.ts`
  - keeps ordered `runtimePendingInputQueue` chunks for mixed `text / binary / text` buffering
  - preserves legacy text preview via `runtimePendingInput`
- `packages/sdkwork-terminal-shell/src/index.tsx`
  - forwards direct binary input to `writeLocalShellInputBytes(...)`
  - flushes queued runtime input by ordered chunk, not by flat string only
  - keeps runtime overlay able to describe queued binary input
- `src-tauri/src/lib.rs`
  - adds `desktop_local_shell_session_input_bytes`
- `crates/sdkwork-terminal-pty-runtime/src/lib.rs`
  - adds `write_input_bytes(...)`
