# 2026-04-11 v0.2.40 Runtime-Node HTTP Input Coverage

## Scope

- Runtime-node public HTTP input API
- Split command text plus standalone `Enter`
- Backspace correction before execution
- Server-mode parity with host and PTY runtime coverage

## Changed

- `crates/sdkwork-terminal-runtime-node/tests/runtime_node_http_test.rs`
  - Added an HTTP integration test that posts command text first and `\r` second, then waits for replay output to confirm real shell execution completed.
  - Added an HTTP integration test that posts a mistyped command first, then sends `DEL` backspace correction plus `Enter`, confirming the `/input` route preserves terminal-style line editing.
  - Switched Windows shell-ready gating from late subscription timing to replay polling, so the test no longer misses prompt events that were emitted before the HTTP test subscribed.
  - Increased replay polling coverage from the first 64 entries to a larger window so verbose PowerShell prompt and syntax-highlighting output cannot hide the final assertion token.

## Fixed

- Closed the last uncovered interaction layer for runtime-node input fidelity: PTY runtime and host tests already passed, but the public HTTP surface did not yet have equivalent regression protection.
- Reduced the chance that server deployments would regress on real typed-shell behavior even though lower layers remained green.
- Fixed a Windows-only false-negative path in the HTTP tests where shell startup output could arrive before the test subscribed to live events.
- Fixed a bounded replay-read issue where the expected command output could fall outside the first replay window under noisy PowerShell output.

## Verified

- `cargo test -p sdkwork-terminal-runtime-node --test runtime_node_http_test -- --nocapture`
- `node --experimental-strip-types --test tests/shell-runtime.test.ts tests/terminal-view-driver.test.ts`

## Notes

- This increment adds coverage only. It does not change package versions, Tauri app versions, or runtime protocol shapes.
- The next highest-value work remains front-end xterm interaction hardening for cursor placement, focus recovery, and tab-switch behavior on the desktop surface.
