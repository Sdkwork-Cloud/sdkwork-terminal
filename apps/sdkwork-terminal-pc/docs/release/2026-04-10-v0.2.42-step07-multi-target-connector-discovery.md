# 2026-04-10 v0.2.42 Step 07 Multi-Target Connector Discovery

## Summary

This round advances Step 07 from a single-authority discovery baseline to a real multi-target connector inventory. Desktop discovery now surfaces multiple SSH hosts, Docker containers, and Kubernetes pods, while preserving configured-authority priority and deduplication. `remote-runtime` remains read-only and outside the live desktop attach path.

## Changed

- `crates/sdkwork-terminal-resource-connectors/src/lib.rs`
  - `discover_connector_execution_targets(...)` now returns multi-target inventory for `ssh / docker-exec / kubernetes-exec`.
  - SSH discovery now merges `SDKWORK_TERMINAL_SSH_AUTHORITY` with multiple valid `Host` aliases from `~/.ssh/config`.
  - Docker discovery now merges `SDKWORK_TERMINAL_DOCKER_AUTHORITY` with multiple running container names from `docker ps --format {{.Names}}`.
  - Kubernetes discovery now merges `SDKWORK_TERMINAL_KUBERNETES_AUTHORITY` with multiple pod authorities from the current context.
  - Configured authorities stay first, duplicates are removed, and placeholder fallback behavior remains for missing inventory or missing tooling.
- `src-tauri/src/lib.rs`
  - Desktop execution-target catalog test coverage now expects multi-target connector inventory instead of one authority per connector.
- `tests/desktop-resource-launch.test.ts`
  - Desktop resource-center and connector-menu coverage now locks multiple same-kind connector entries at the product layer.
- `README.md`
- `docs/架构/06-终端会话、运行目标与协议设计.md`
- `docs/step/07-SSH、Docker、Kubernetes与Remote-Runtime连接器落地.md`
  - Updated the current discovery baseline and Step 07 truth to match multi-target inventory.
