# 2026-04-10 v0.2.40 - Desktop Attachment Ack Feedback

## Changed

- Added `desktop_session_attachment_acknowledge` to the desktop bridge so attachment ack is no longer Rust-internal only.
- `packages/sdkwork-terminal-shell` now persists `runtimeAttachmentId` on runtime-backed tabs and acknowledges the latest replay sequence after live event and replay catch-up application.
- This closes the drift where `sessionIndex.attachments.lastAckSequence` could stay pinned to startup values even after the desktop terminal had already consumed newer replay output.

## Risk / Rollback

- This change is best-effort on the frontend hot path: replay rendering does not fail closed if attachment ack write-back fails.
- If rollback is needed, keep `desktop_session_attachment_acknowledge` disabled before removing `runtimeAttachmentId` tracking, otherwise Session Center evidence will drift back to stale attachment acks.
