# 2026-04-11 v0.2.40 Runtime Replay Transcript Cold Cache

## Summary

Runtime tabs no longer maintain a second adapter-backed transcript during PTY replay. The runtime path now treats `runtimeTerminalContent` plus xterm as the only replay surface, while the adapter snapshot stays empty/cold for runtime-rendered tabs.

## Changes

- Cleared the adapter transcript when `bindTerminalShellSessionRuntime(...)` attaches a live runtime session.
- Stopped `applyTerminalShellReplayEntries(...)` from mirroring runtime replay output into the adapter transcript for runtime-stream tabs.
- Kept raw replay bytes, exit markers, truncation behavior, and cursor/state updates on `runtimeTerminalContent`.
- Updated shell tests to assert that runtime tabs preserve replay in raw content only and do not rebuild adapter transcript lines.

## Impact

- Cuts duplicate terminal-core line building during runtime replay.
- Lowers snapshot churn and memory pressure during `new tab`, shell bootstrap, and high-output sessions.
- Preserves reactivation fidelity because xterm catch-up still rehydrates from raw runtime content.
