# 2026-04-11 v0.2.40 Terminal Bootstrap Performance and Theme

## Scope

- Desktop `new tab` responsiveness
- Runtime-control effect churn
- Windows Terminal-style dark theme alignment
- xterm surface/background consistency

## Changed

- `packages/sdkwork-terminal-shell/src/model.ts`
  - Added `resolveTerminalShellRuntimeBootstrapRequestFromTab(...)` so bootstrap requests can be built directly from the tab snapshot used by the UI.
  - Added tab-snapshot reference caching keyed by immutable tab-state instances plus `active/closable` shape, so unchanged tabs can be reused across repeated shell snapshot reads.
- `packages/sdkwork-terminal-shell/src/index.tsx`
  - Added narrow effect keys for runtime binding sync, retry scheduling, bootstrap launch, subscription/poll orchestration, and pending-input flush.
  - Removed the bootstrap path's dependency on full-shell-state request lookup.
  - Added `tabId -> latest snapshot` lookup helpers for runtime input, restart, and command submission so memoized stage callbacks no longer depend on render-time tab objects.
  - Memoized `TerminalStage` so inactive panels can skip rerender when their tab snapshot reference is unchanged.
  - Moved command-submit dedupe bookkeeping from React state to a `Set` ref so submit/restart control flow does not re-render the full shell tree on its own.
  - Recolored app chrome to a charcoal tab/header palette and switched terminal-stage surfaces to a deeper near-black background.
- `packages/sdkwork-terminal-infrastructure/src/index.ts`
  - Updated the xterm base theme background, cursor accent, selection, and foreground to match the darker terminal stage.
- `tests/shell-tabs.test.ts`
  - Added coverage that runtime bootstrap requests can be resolved from a tab snapshot without the full shell state.
  - Added coverage that unchanged tab snapshots retain reference identity across repeated reads and unrelated tab updates.
- `tests/shell-app-render.test.ts`
  - Added source-contract coverage for the narrowed bootstrap effect key, latest-tab lookup helpers, memoized terminal stage, and the dark theme tokens.
- `tests/terminal-view-driver.test.ts`
  - Updated xterm driver theme assertions to the new near-black surface palette.

## Fixed

- Reduced `new tab` stalls caused by runtime-control effects being re-evaluated on broad tab snapshot churn from unrelated PTY or command-buffer updates.
- Reduced repeated bootstrap scans during live terminal interaction by narrowing effect triggers to the specific runtime lifecycle fields they actually consume.
- Reduced inactive panel rerenders by reusing unchanged tab snapshot objects and memoizing the terminal stage.
- Reduced render-time closure staleness risk by resolving current tab state at event time instead of reading render-captured tab objects.
- Reduced extra render passes during command submission and runtime restart bookkeeping by removing non-visual pending-submit state from React render flow.
- Fixed the visual mismatch between the header tabs and the terminal body by aligning them to gray-black chrome plus deep-black content.
