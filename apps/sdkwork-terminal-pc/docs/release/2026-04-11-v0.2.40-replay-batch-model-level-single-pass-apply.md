# 2026-04-11 v0.2.40 Replay Batch Model-Level Single-Pass Apply

## Summary

Desktop runtime replay flushes no longer rewrap shell state once per queued session batch.

## Changes

- Added `applyTerminalShellReplayBatches(...)` in the shell model for multi-batch replay application.
- Moved shared replay-application logic into one tab-level helper reused by both single-batch and multi-batch paths.
- Updated the desktop replay flush path to call the model-level batch helper directly instead of chaining replay updates with `reduce(...)`.
- Added regression coverage for stale-batch state identity, mixed stale/fresh multi-tab replay application, and the source contract for model-layer batch application.

## Impact

- Reduces repeated shell-state and tab-array cloning when several PTY sessions flush replay in the same frame.
- Lowers replay flush pressure during multi-tab startup, catch-up repair, and burst output windows.
- Keeps replay semantics stable while tightening the highest-frequency multi-session apply path.
