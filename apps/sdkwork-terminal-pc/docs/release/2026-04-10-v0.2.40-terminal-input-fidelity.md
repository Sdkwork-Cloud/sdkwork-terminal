# 2026-04-10 v0.2.40 Terminal Input Fidelity

## Summary

本轮聚焦真实 terminal 主链，不再只做视觉外观。重点收口 `terminal 无法输入` 风险、xterm viewport 质感，以及 Rust PTY 与前端桥接的架构边界。

## Changed

- `packages/sdkwork-terminal-shell/src/index.tsx`
  - active terminal stage 在 attach 完成后主动 focus，降低首屏输入失焦问题。
  - Desktop tab 在 runtime `binding` 期间先缓存 raw input，session 进入 `running` 后再顺序冲刷到 PTY，避免新 tab 启动期按键丢失。
  - Desktop terminal stage 在 `binding / queued input` 阶段新增原生状态 overlay，明确呈现 `Starting shell / Input queued`，避免用户误判为卡死或无法输入。
  - Desktop terminal stage 在 `failed / exited` 阶段新增 `Restart shell` 恢复动作，允许同 tab 原位重启真实 shell，而不是只能关闭重开。
  - tabs/header/window controls/tab close 增加 hover feedback。
  - xterm viewport scrollbar 收口为更细、更暗、更低噪音的终端风格。
- `packages/sdkwork-terminal-shell/src/model.ts`
  - 新增 tab 级 runtime restart 重置逻辑，重启时会清理旧 session id、cursor、pending input、runtime transcript 与 exit code，同时保留 tab 身份、profile 与 viewport。
- `packages/sdkwork-terminal-shell/src/model.ts`
  - Desktop 默认 shell 增加宿主平台兜底；浏览器 `navigator` 不可用时，Windows 主机仍默认 PowerShell。
- `packages/sdkwork-terminal-infrastructure/src/index.ts`
  - xterm driver 增加 `fontSize: 14`、`lineHeight: 1.25`、`letterSpacing: 0.15`、`allowTransparency: true`、`scrollback: 5000`。
  - xterm Unicode provider 改为显式激活 `11` 版本，消除 `Unicode11Addon.version` 类型漂移引发的 `pnpm typecheck` 失败。
- 文档回写：
  - `docs/架构/README.md`
  - `docs/架构/08-渲染、兼容性与终端体验设计.md`
  - `docs/step/05-Terminal-Core与渲染交互基线落地.md`

## Architecture Conclusion

- Rust PTY 主链目前可用，`sdkwork-terminal-pty-runtime` 已证明本地交互式 shell session 可 create / write / resize / terminate。
- 当前更大的用户感知风险仍在前端终端壳层：焦点、binding 阶段输入缓存、输入回显体验、viewport 质感、header 交互必须继续按终端产品标准打磨。
- 当前“启动中无反馈”的认知断层已收口到 terminal overlay，但失败态、恢复态与跨平台人工 smoke 证据仍未闭环。
- 当前失败后的人工恢复动作已具备，但自动恢复策略与三平台手工 smoke 证据仍未闭环。
- 专业 terminal 的正确架构仍是：
  - `xterm onData -> desktop bridge -> Tauri IPC -> Rust PTY writer -> shell echo -> runtime event/replay -> xterm writeRaw`
