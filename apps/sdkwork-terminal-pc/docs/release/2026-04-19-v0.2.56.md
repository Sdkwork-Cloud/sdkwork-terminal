# 2026-04-19 v0.2.56

## Release Strategy

- This release turns `@sdkwork/terminal-shell` from a workspace-facing source package into a publishable third-party component package.
- The goal is to lock a clean distribution boundary so external hosts consume prebuilt bundles, stable declarations, and explicit stylesheet assets instead of internal monorepo source paths.

## Wave / Step

- Wave B
- Component packaging hardening
- Terminal shell third-party distribution standardization

## Changed

- Reworked `@sdkwork/terminal-shell` to publish only `dist/` plus `README.md`, with explicit ESM exports for the root module and `./integration`.
- Added package-owned declaration files that describe the public shell runtime client and host integration contracts without leaking workspace-only infrastructure types.
- Added a dedicated package build pipeline that emits JavaScript bundles, declaration files, bundled terminal CSS assets, and a Windows-safe `prepack` launcher.
- Added review evidence for the component package distribution contract and extended package contract tests to lock the new export and prepack behavior.

## Fixed

- Fixed the package boundary defect where third-party consumers still had to resolve `src/*.ts(x)` files from the published shell package.
- Fixed public API leakage where shell contract types depended on monorepo workspace packages rather than a package-owned integration surface.
- Fixed `pnpm pack` on Windows where the lifecycle script could fail because `node` was unavailable in the lifecycle `PATH`.
