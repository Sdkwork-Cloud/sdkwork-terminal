# 2026-04-19 v0.2.54

## Release Strategy

- This release advances the shipped `v0.2.53` line with a narrow release-engineering correction: remove the remaining Node20-based GitHub Actions dependencies from the release pipeline.
- The objective is not product-surface change; it is to keep the desktop packaging workflow operationally clean and aligned with GitHub's current JavaScript action runtime direction.

## Wave / Step

- Wave B
- Release engineering hardening
- GitHub release workflow Node24 alignment

## Changed

- Upgraded `actions/upload-artifact` from `v4` to `v7` in the reusable release workflow.
- Upgraded `actions/download-artifact` from `v4` to `v8` in the reusable release workflow.
- Upgraded `softprops/action-gh-release` from `v2` to `v3` in the reusable release workflow.
- Updated the workflow contract test to lock the new action versions and explicitly reject the previous deprecated versions.

## Fixed

- Fixed the remaining release-pipeline deprecation warnings caused by Node20-targeted artifact and release actions.
- Fixed a release-maintenance risk where future GitHub runner policy changes could turn a warning-only condition into a broken publishing path.
