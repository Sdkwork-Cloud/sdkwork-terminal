# 2026-04-10 v0.2.49 Step 07 Web Shell Runtime Binding

## Summary

This round closes the missing product slice between the Step 07 `remote-runtime` web bridge foundation and the actual web shell surface. `apps/web` now boots `ShellApp` with a real web runtime client and a runtime target descriptor, while `ShellApp` itself can carry `remote-runtime` bootstrap requests as first-class tab truth.

## Changed

- `packages/sdkwork-terminal-shell/src/model.ts`
  - Added tab-level `remote-runtime` bootstrap cloning and request resolution.
- `packages/sdkwork-terminal-shell/src/index.tsx`
  - Added `webRuntimeClient` and `webRuntimeTarget`.
  - Unified desktop/web session hot paths for bootstrap, replay, input, resize, terminate, and stream subscribe.
  - Kept desktop-only `detachSessionAttachment` and `acknowledgeSessionAttachment` boundaries unchanged.
  - Web first tab and new tabs now enter a runtime-backed terminal path when `webRuntimeTarget` is configured.
  - Web still falls back to the local prompt only when runtime target configuration is absent.
- `apps/web/src/App.tsx`
  - Added `createWebRuntimeBridgeClient(...)`.
  - Added `VITE_TERMINAL_RUNTIME_BASE_URL / WORKSPACE_ID / AUTHORITY / TARGET / WORKING_DIRECTORY` bootstrap wiring.
- `apps/web/.env.example`
  - Added minimal runtime bootstrap environment documentation.
- `apps/web/package.json`
  - Added `@sdkwork/terminal-infrastructure`.
- `tests/shell-app-render.test.ts`
  - Added source-shape coverage for the new web runtime wiring.
