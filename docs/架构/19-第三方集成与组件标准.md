# 第三方集成与组件标准

## 1. 目标

本标准定义 `sdkwork-terminal` 作为第三方可集成组件时的公开边界、样式接入方式、宿主职责和禁止事项，避免外部应用依赖内部实现细节。

## 2. 公开入口

- 组件根入口：`@sdkwork/terminal-shell`
- 集成入口：`@sdkwork/terminal-shell/integration`
- 样式入口：`@sdkwork/terminal-shell/styles.css`
- Runtime Bridge 入口：`@sdkwork/terminal-infrastructure`

## 3. 组件层级

### 3.1 根组件

- `ShellApp`
- `ShellAppProps`
- `ShellAppDesktopRuntimeClient`
- `ShellAppWebRuntimeClient`

### 3.2 第三方宿主包装组件

- `DesktopShellApp`
- `WebShellApp`

### 3.3 Web 集成辅助

- `createBrowserClipboardProvider`
- `createWebRuntimeTargetFromEnvironment`

## 4. 宿主职责

### 4.1 Desktop 宿主

- 提供桌面 runtime client
- 提供桌面窗口控制器
- 按需提供工作目录选择器、Session Center、Connector 菜单等产品级能力

### 4.2 Web 宿主

- 提供 web runtime client
- 提供 `WebRuntimeTarget`
- 提供浏览器剪贴板能力

## 5. 样式标准

- 必须通过 `@sdkwork/terminal-shell/styles.css` 引入样式
- 不允许依赖 `ShellApp` 模块内部的样式副作用导入
- Desktop 和 Web 入口都必须显式引入该样式入口

## 6. 禁止事项

- 禁止从 `packages/sdkwork-terminal-shell/src/*` 深层导入
- 禁止从第三方宿主直接复制 browser clipboard 逻辑或 web runtime target 解析逻辑
- 禁止把 `src-tauri` 细节暴露为第三方公共 API
- 禁止把内部 feature model 当作公开集成契约

## 7. 验收标准

- 包导出中存在 `./integration` 和 `./styles.css`
- Workspace alias 支持 `@sdkwork/terminal-shell/integration`
- `apps/web` 和 `apps/desktop` 使用公开 wrapper 组件
- `apps/web` 不再本地重写浏览器剪贴板逻辑
- README 和 review 文档能够说明第三方接入规则

## 8. 包分发标准

- `@sdkwork/terminal-shell` 发布内容限定为 `README.md` 与 `src/`，避免把临时文件或宿主私有资产带入第三方分发包
- `@sdkwork/terminal-shell/styles.css` 与 `src/shell-app.css` 必须声明为 side effect，避免消费端 tree shaking 误删终端样式
- `pnpm pack` 后不得保留 `workspace:*` 依赖范围，必须固化为当前版本线的显式版本号
- 第三方宿主接入时，`@sdkwork/terminal-shell` 与相关 `@sdkwork/terminal-*` 依赖必须保持同一发布版本线
