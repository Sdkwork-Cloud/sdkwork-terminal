# 2026-04-18 v0.2.51

## Release Strategy

- This release consolidates unpublished release candidates `v0.2.41` through `v0.2.50`, the `2026-04-11 v0.2.40-*` terminal/runtime increment notes, and the current desktop working-directory picker launch flow into one formal version.
- No successful Git tag or authenticated GitHub release baseline was available in this session. The local repository history contains no tags and only one visible commit, so this release summary is rebuilt from `docs/release/`, current workspace changes, and local verification evidence.

## Changed

- Desktop AI CLI launch profiles now open a native system folder picker before launching `Codex CLI`, `Claude Code`, `Gemini CLI`, or `OpenCode CLI`.
- `ShellApp` now seeds the selected `workingDirectory` into both the initial tab state and the `local-process` runtime bootstrap request, so the CLI session starts directly in the chosen workspace.
- The desktop host now exposes an app-owned `desktop_pick_working_directory` bridge command backed by `tauri-plugin-dialog`, and the desktop permission set now explicitly grants that command.
- The desktop app/session-center surface now carries forward the unpublished session-center refresh, replay preload, connector intent, and resource-catalog work that accumulated after `0.2.40`.
- The terminal shell/runtime surface now carries forward the unpublished runtime-stage split, clipboard routing, replay batching, runtime-derived-state caching, xterm mutation serialization, bootstrap/input gating cleanup, and Windows PTY fidelity fixes captured in the `2026-04-11 v0.2.40-*` notes.
- The Step 07 connector and remote-runtime line now formally includes the unpublished `v0.2.41` through `v0.2.50` work:
  - desktop connector reattach alignment
  - multi-target connector discovery
  - connector smoke scope alignment
  - connector batch smoke matrix/template/preflight/execution-plan tooling
  - remote-runtime web bridge
  - runtime-node host core

## Fixed

- Fixed the desktop product gap where selecting an AI CLI still dropped the user into a generic shell that required manual command entry.
- Fixed explicit local-process tab bootstrap so selected directories are visible before runtime binding and preserved across bootstrap request resolution.
- Fixed Windows user-facing working-directory presentation by stripping raw device prefixes such as `\\?\` before surfacing the path in the product UI.
- Fixed the desktop bridge ACL surface so folder picking is part of the app-owned command set rather than an implicit or missing permission.
- Fixed multiple unpublished desktop/runtime regressions captured in the carried-forward notes: PTY input gating, replay batching pressure, tab update churn, runtime/fallback stage drift, focus and paste-path drift, and Windows interactive probe alignment.
