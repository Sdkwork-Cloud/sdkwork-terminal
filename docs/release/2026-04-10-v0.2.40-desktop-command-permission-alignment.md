# 2026-04-10 v0.2.40 Desktop Bridge Command Permission Alignment

## Summary

This round stays inside the current desktop host line and fixes a runtime ACL regression: the desktop bridge invoked `snake_case` command names such as `desktop_local_shell_session_create`, while the autogenerated Tauri permission manifests only exposed kebab-case command entries. The result was a false `not allowed. Command not found` failure even though the Rust command was registered.

## Changed

- `src-tauri/permissions/desktop-host.toml`
  - Added the app-owned `desktop-host-commands` permission set.
  - Declared every desktop bridge command with its invoke-visible `snake_case` name.
- `src-tauri/capabilities/default.json`
  - Added `desktop-host-commands` to the main desktop capability bound to the `main` window.
- `tests/desktop-tauri-permissions.test.ts`
  - Added a regression test that fails if the main capability stops mounting the manual permission set or if any desktop bridge command loses its `snake_case` permission entry.
