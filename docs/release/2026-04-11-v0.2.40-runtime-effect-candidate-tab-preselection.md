# 2026-04-11 v0.2.40 Runtime Effect Candidate Tab Preselection

## Changed

- `packages/sdkwork-terminal-shell/src/index.tsx` now precomputes `runtimeBootstrapCandidateTabs` and `runtimePendingInputTabs` inside `createRuntimeDerivedState(...)`.
- Runtime bootstrap and pending-input flush effects now iterate those candidate lists instead of rescanning `snapshot.tabs`.
- `tests/shell-app-render.test.ts` now freezes the candidate-tab preselection contract.

## Fixed

- Reduced repeated full-array scanning after runtime effect keys changed, especially in workspaces where most tabs are already stable running sessions.
- Reduced one more source of main-thread overhead in `new tab`, retry, and pending-input flush paths.
- Kept runtime behavior unchanged while narrowing effect work to the tabs that actually need action.
