# 2026-04-11 v0.2.40 Desktop PTY Input Unblock and Helper Textarea Hardening

## Changed

- `packages/sdkwork-terminal-shell/src/runtime.ts` now lets desktop PTY input bypass and flush immediately after the session reaches `running`, instead of waiting for the first repaint/output frame.
- `packages/sdkwork-terminal-shell/src/index.tsx` now forces xterm's helper textarea off-screen with fixed positioning, 1px sizing, zero opacity, and disabled pointer events.
- `tests/shell-runtime.test.ts` and `tests/shell-app-render.test.ts` now lock both the interactive-input and helper-textarea contracts.

## Fixed

- Fixed the false "terminal is open but commands do nothing" state caused by first-frame input gating on running desktop sessions.
- Fixed a visible-caret regression path where xterm's hidden helper textarea could leak into the top-left corner of the terminal surface.
