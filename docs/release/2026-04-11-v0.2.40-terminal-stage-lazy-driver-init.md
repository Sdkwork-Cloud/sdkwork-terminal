# 2026-04-11 v0.2.40 Terminal Stage Lazy Xterm Driver Initialization

## Scope

- `new tab` render cost reduction
- Tab-switch responsiveness
- Per-render xterm driver factory churn removal
- Terminal stage stability hardening

## Changed

- `packages/sdkwork-terminal-shell/src/index.tsx`
  - Replaced eager `useRef(createXtermViewportDriver())` with lazy `ref` initialization inside `TerminalStage`.
  - Bound subsequent terminal-stage operations to the already-initialized stable driver handle for the current tab.
- `tests/shell-app-render.test.ts`
  - Added regression coverage that the shell stage keeps lazy driver initialization and rejects the eager driver-factory pattern.

## Fixed

- Fixed `TerminalStage` executing the xterm driver factory on every render even though only the first driver instance was retained.
- Reduced avoidable allocation and closure setup during frequent React updates from runtime replay, runtime status, search, resize, and command-buffer changes.
- Reduced one more contributor to `new tab` stalls and sluggish tab switching in the desktop terminal surface.
