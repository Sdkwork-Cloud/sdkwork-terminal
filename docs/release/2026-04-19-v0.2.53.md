# 2026-04-19 v0.2.53

## Release Strategy

- This release advances the shipped `v0.2.52` line by converting the terminal shell into a documented third-party integration surface instead of an internal-only assembly entrypoint.
- The scope is to make external desktop and web hosts consume the same supported package contract that the in-repo apps consume, while preserving explicit style loading and release-time package integrity.

## Wave / Step

- Wave B
- Step 07 continued
- Terminal shell third-party integration surface and packaging standardization

## Changed

- Added stable public shell host contracts in `@sdkwork/terminal-shell`, including `ShellAppProps`, explicit desktop/web runtime client type exports, and working-directory picker options.
- Added `@sdkwork/terminal-shell/integration` with `DesktopShellApp`, `WebShellApp`, `createBrowserClipboardProvider`, and `createWebRuntimeTargetFromEnvironment`.
- Added `@sdkwork/terminal-shell/styles.css` and switched both desktop and web hosts to load styles through the explicit public stylesheet entrypoint.
- Added package distribution metadata with `files` and `sideEffects`, plus package README and architecture/review documentation for third-party integration rules.

## Fixed

- Fixed a public-surface gap where desktop and web hosts were still proving internal assembly details instead of a stable package-owned integration contract.
- Fixed bundler fragility by removing implicit in-component shell style imports and by preserving terminal styles as explicit package side effects.
- Fixed workspace alias precedence so the stylesheet subpath resolves correctly during Vite builds instead of being captured by the root package alias.
- Fixed external-host duplication risk by centralizing browser clipboard and runtime-target environment parsing into the shell integration layer.

## Risks And Rollback

- Third-party integration is now standardized at the package boundary, but the shell package still expects sibling `@sdkwork/terminal-*` packages to stay on the same released version line.
- Cross-platform release publication still depends on the GitHub Actions packaging matrix after the `v0.2.53` tag is pushed.
- Rollback target: `v0.2.52`.
