# 2026-04-19 v0.2.57

## Release Strategy

- This release closes the last external-integration verification gap for `@sdkwork/terminal-shell`.
- The goal is to prove that the published tarball can be consumed by a real third-party React/Vite host, not only by workspace-local package aliases.

## Wave / Step

- Wave B
- Component packaging hardening
- Third-party consumer tarball smoke baseline

## Changed

- Added a minimal third-party consumer fixture under `tests/fixtures/third-party-shell-consumer`.
- Added an automated smoke test that packs `@sdkwork/terminal-shell`, extracts the tarball into an isolated consumer `node_modules`, then runs TypeScript and Vite build verification against that external host.
- Updated package documentation and package-distribution review evidence so the public shell contract now requires packed-consumer proof in addition to internal package build and pack verification.

## Fixed

- Fixed the remaining release-proof gap where `@sdkwork/terminal-shell` could be packed successfully but lacked evidence that a real external host could consume the tarball without workspace aliases.
- Fixed a standards gap where third-party integration validation depended on monorepo-local assumptions instead of a consumer-side smoke baseline.

## Risks And Rollback

- This release tightens verification and distribution standards. The main risk is longer local verification time because the packed consumer smoke now performs a real tarball build-and-consume cycle.
- The proof target is the successful packed-consumer smoke test plus the GitHub-hosted `v0.2.57` release run.
- Rollback target: `v0.2.56`.
