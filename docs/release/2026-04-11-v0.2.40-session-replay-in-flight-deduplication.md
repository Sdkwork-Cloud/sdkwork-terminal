# 2026-04-11 v0.2.40 Session Replay In-Flight Deduplication

## Changed

- `packages/sdkwork-terminal-shell/src/index.tsx` now keeps a per-session in-flight set for replay repair requests and skips starting a new `sessionReplay` call while one is already pending for the same session.
- The in-flight marker is cleared during unmount and in `finally` after each replay repair request settles.
- `tests/shell-app-render.test.ts` now freezes the replay in-flight deduplication contract.

## Fixed

- Reduced overlapping replay pulls that could stack up when timer-based catch-up and subscription-triggered catch-up targeted the same session at the same time.
- Reduced one more source of desktop load and UI contention during `new tab`, tab switching, and busy-output recovery paths.
- Kept replay repair behavior intact for different sessions and for later retries after an earlier request completes.
