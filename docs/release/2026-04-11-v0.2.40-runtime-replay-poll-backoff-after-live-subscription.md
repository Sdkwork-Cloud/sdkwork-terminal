# 2026-04-11 v0.2.40 Runtime Replay Poll Backoff After Live Subscription

## Changed

- `packages/sdkwork-terminal-shell/src/runtime.ts` now returns a `1500ms` steady-state replay poll interval once every bound runtime session has a healthy live subscription.
- Recovery polling remains `120ms` while subscriptions are missing, unavailable, or marked failed.
- `tests/shell-runtime.test.ts` now freezes the steady-state poll-backoff contract.

## Fixed

- Reduced the steady-state replay polling load that kept competing with real PTY subscriptions in desktop tabs even after the live stream path was already healthy.
- Reduced one more source of `new tab`, tab-switch, and multi-tab output pressure by lowering unnecessary `sessionReplay` traffic.
- Kept fast catch-up behavior for degraded subscription states instead of weakening the recovery path.
