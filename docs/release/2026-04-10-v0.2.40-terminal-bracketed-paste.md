# 2026-04-10 v0.2.40 Terminal Bracketed Paste

## Summary

本轮继续停留在 `Wave B / Step 05`，收口 clipboard paste 热路径。目标是避免剪贴板文本绕过 xterm，破坏 bracketed paste 等真实 terminal 语义。

## Changed

- `packages/sdkwork-terminal-infrastructure/src/index.ts`
  - `XtermViewportDriver` 新增 `paste(text)` 能力。
  - driver 优先调用 xterm 原生 `terminal.paste(text)`；仅在原生能力缺失时回退到 input listener。
- `packages/sdkwork-terminal-shell/src/index.tsx`
  - terminal 内快捷键粘贴改为走 `driverRef.current.paste(safeText)`。
  - tab 级菜单 `Paste` 新增 viewport paste handler 注册链路，优先走当前 tab 的 xterm driver。
  - 移除剪贴板文本直接注入 `props.onViewportInput(safeText)` 的主路径。
