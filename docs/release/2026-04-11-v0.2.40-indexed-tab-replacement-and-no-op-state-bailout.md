# 2026-04-11 v0.2.40 Indexed Tab Replacement And No-Op State Bailout

## Summary

Single-tab shell updates no longer rebuild the full tab array, and no-op tab updates now preserve shell state identity.

## Changes

- Added `getTabIndexOrThrow(...)` to resolve one tab index and reuse that lookup for replacement.
- Replaced `withTab(...)` full-array `map(...)` rebuilding with `slice()` plus indexed assignment.
- Added a shell-state identity bailout so stale replay batches can return the original state object.
- Added regression coverage for both the state-identity contract and the indexed replacement source contract.

## Impact

- Reduces per-update callback churn on hot paths such as replay apply, runtime input, resize, and tab-local state transitions.
- Reduces avoidable React work when replay or command flows resolve to a true no-op.
- Keeps targeted tab updates precise without changing shell behavior or tab ordering semantics.
