# 2026-04-21 v0.2.58

## Release Strategy

- This release closes the desktop CLI launch standardization loop and the TUI scrollback regression that still affected Codex-style interactive sessions.
- The goal is to make `@sdkwork/terminal-shell` a cleaner component surface for desktop hosts while preserving release-grade runtime behavior in packaged Windows builds.

## Wave / Step

- Wave B
- Desktop terminal product hardening
- Project-aware CLI launch and TUI transcript recovery

## Changed

- Refactored `@sdkwork/terminal-shell` into explicit launch, runtime, overlay, chrome, and state modules so the public shell surface is easier to integrate and review.
- Added project-aware desktop CLI launch flows: a single supplied project opens directly, multiple projects open a built-in picker dialog, and missing project catalogs fall back to the native directory picker.
- Added recent-project persistence helpers and public shell contract events for launch-project activation, removal, and collection clearing so hosts can treat project launch as first-class product state.

## Fixed

- Fixed the runtime regression where TUI control sequences such as alternate-screen enter/exit, synchronized output, mouse-reporting toggles, and destructive redraw commands were still reaching xterm as raw escape sequences, which broke scrollback for Codex and similar TUIs.
- Fixed replay-gap recovery so full replay repair now restores a linearized transcript view instead of reintroducing raw TUI control frames after session rebuild.
- Fixed the desktop release confidence gap by re-running full local Windows release build and launch smoke validation against the refactored shell/runtime surface.

## Risks And Rollback

- The main risk is behavioral drift across the newly split shell modules and project-launch flows, especially around desktop-only launch orchestration and replay hydration.
- The proof target is the successful local `pnpm verify:windows-release` run plus the GitHub-hosted `v0.2.58` release publication.
- Rollback target: `v0.2.57`.
