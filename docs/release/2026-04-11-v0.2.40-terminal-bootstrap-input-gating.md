# 2026-04-11 v0.2.40 Terminal Bootstrap Input Gating

## Scope

- Desktop PTY bootstrap timing
- Preloaded CLI tab commands
- Pending runtime-input flush policy
- Windows Terminal-style first-command stability

## Changed

- `packages/sdkwork-terminal-shell/src/runtime.ts`
  - Added `shouldFlushTerminalRuntimeInputQueue(...)` so queue flushing is governed explicitly instead of piggybacking on the live-write bypass rule.
  - Desktop runtime input now waits for the first PTY frame before flushing normal text, while protocol replies such as CPR/DSR still pass through early.
- `packages/sdkwork-terminal-shell/src/model.ts`
  - Added `queueTerminalShellTabBootstrapCommand(...)` so a bootstrap command is converted into buffered runtime input and the visible command buffer is cleared in one state transition.
- `packages/sdkwork-terminal-shell/src/index.tsx`
  - Replaced direct PTY writes for bootstrap command text with the shared pending-input queue path.
  - Replaced eager desktop queue flush on `runtimeState === running` with guarded flush logic that honors `runtimeStreamStarted`.
- `tests/shell-runtime.test.ts`
  - Added coverage for the desktop pre-stream flush gate and the web immediate-flush path.
- `tests/shell-tabs.test.ts`
  - Added coverage that a preloaded CLI command becomes queued runtime input instead of a direct PTY write.
- `tests/shell-app-render.test.ts`
  - Locked the ShellApp source contract so bootstrap commands stay queued and the flush gate remains present.

## Fixed

- Fixed desktop tabs being able to flush queued command text before the shell had emitted its first real frame.
- Fixed preloaded CLI tabs like `codex` and `claude` bypassing the same runtime-input buffering rules as real keyboard input.
- Reduced startup-time input instability that can surface as broken first-command execution or visually confusing early echo on Windows-heavy shells.

## Verified

- `node --experimental-strip-types --test tests/shell-runtime.test.ts tests/shell-tabs.test.ts tests/shell-app-render.test.ts`
- `pnpm typecheck`

## Notes

- This increment does not change package versions, runtime protocol shapes, or Tauri versioning.
- The next highest-value work remains desktop xterm interaction hardening around cursor placement, tab-switch focus recovery, and viewport geometry polish.
