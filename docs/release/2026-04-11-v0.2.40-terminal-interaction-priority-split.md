# 2026-04-11 v0.2.40 Terminal Interaction Priority Split

## Changed

- `packages/sdkwork-terminal-shell/src/index.tsx` now separates shell state updates into direct user-interaction updates and deferred runtime updates.
- Immediate tab actions, activation changes, and command editing continue through `updateShellState(...)`.
- Runtime replay, async bootstrap transitions, and pending-input flush completion now flow through `updateShellStateDeferred(...)`.
- `tests/shell-app-render.test.ts` now freezes this split as a source contract.

## Fixed

- Reduced `new tab` and tab-switch sluggishness caused by routing every shell state change through `startTransition(...)`.
- Reduced the risk that async PTY replay and bootstrap bookkeeping would delay visible response to direct terminal interactions.
