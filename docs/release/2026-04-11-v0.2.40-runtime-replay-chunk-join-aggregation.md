# 2026-04-11 v0.2.40 Runtime Replay Chunk Join Aggregation

## Summary

Runtime replay batches now aggregate raw output chunks first and append them to `runtimeTerminalContent` once per batch.

## Changes

- Added `runtimeOutputChunks` aggregation inside `applyTerminalShellReplayEntries(...)`.
- Replaced repeated per-entry runtime string concatenation with one `join("")` append per replay batch.
- Added source-contract coverage to prevent regression to the older per-entry append path.

## Impact

- Reduces string-copy churn when replay batches contain many PTY output entries.
- Improves catch-up and burst replay efficiency without changing terminal semantics.
- Keeps raw VT output, exit markers, and truncation behavior intact.
