# 2026-04-10 v0.2.40 Terminal Shell Runtime UI Refinement

## Scope

- Windows Terminal 风格 tabs/header 行为校正
- Desktop PTY tab 切换后的终端保活与真实终端渲染收口
- 终端舞台贴边显示，移除桌面模式伪输入栏干扰

## Changed

- `packages/sdkwork-terminal-shell/src/model.ts`
  - 新增 `resolveTerminalTabActionInlineWidth(...)`，用于按“可恢复的 inline 宽度”判断 tab action 是否需要 dock 到右侧。
  - 保持 `shouldDockTerminalTabActions(...)` 作为最终宽度决策入口，便于 header 布局规则长期稳定复用。
- `packages/sdkwork-terminal-shell/src/index.tsx`
  - header 宽度测量改为在 dock 状态下补回 action 宽度，修复 `+ / dropdown` 过早固定到右侧的问题。
  - `tabShellStyle(...)` 改为 `flex: "1 0 0"`，让 dock 后 tabs 优先占满剩余可用宽度，而不是被过度压缩。
  - 桌面终端舞台移除 inset，`xterm` 视口改为零 padding 贴边渲染。
  - `showLivePrompt` 收口为仅 `web` fallback 模式显示，桌面 PTY tab 不再叠加伪 prompt UI。
- `tests/shell-tabs.test.ts`
  - 新增 dock 宽度恢复场景测试，锁定“dock 后仍应允许回到 inline”的行为。
- `tests/shell-app-render.test.ts`
  - 新增对零 padding、桌面模式无伪 prompt、dock 后 tab 宽度策略的源码约束。

## Fixed

- 修复 tab 数量较少时，`+ / dropdown` 仍提前停在最右侧的问题。
- 修复 dock 到右侧后 tabs 显示宽度被过度压缩、不能充分利用剩余宽度的问题。
- 修复桌面终端面板四周存在 inset/padding，视觉上不像真实终端窗口的问题。
- 修复桌面 PTY tab 仍叠加 HTML prompt 条的产品偏差，避免与真实 shell 输入反馈冲突。
