# 2026-04-10 v0.2.40 - Step 07 Host Discovery Refresh Baseline

## Summary

- 本轮把 Step 07 `desktop_execution_target_catalog` 从静态演示目录推进到真实 host/config/health discovery baseline，并补齐桌面顶部 profile/connectors dropdown 打开前的 snapshot refresh。

## Changed

- `crates/sdkwork-terminal-resource-connectors`
  - 新增共享 `discover_connector_execution_targets(...)`
  - 冻结 discovery 输入：`workspace_id / home_dir / ssh_authority / docker_authority / kubernetes_authority / remote_runtime_authority`
  - 冻结 discovery 规则：
    - `SSH=SDKWORK_TERMINAL_SSH_AUTHORITY | ~/.ssh/config 首个 Host`
    - `Docker=SDKWORK_TERMINAL_DOCKER_AUTHORITY | docker ps 首个运行容器`
    - `Kubernetes=SDKWORK_TERMINAL_KUBERNETES_AUTHORITY | current-context + 首个 pod`
    - `Remote Runtime=SDKWORK_TERMINAL_REMOTE_RUNTIME_AUTHORITY`
- `src-tauri/src/lib.rs`
  - `desktop_execution_target_catalog` 改为消费共享 Rust discovery 结果
  - host 只保留 local-shell 补充投影与 bridge snapshot 映射
- `packages/sdkwork-terminal-shell/src/index.tsx`
  - 新增 `onBeforeProfileMenuOpen`
  - 顶部 header 在 profile/connectors dropdown 打开前可先触发外部 refresh
- `apps/desktop/src/App.tsx`
  - 现在会在首次加载与 profile menu 打开前刷新 `resourceCenterSnapshot`
