# 2026-04-11 v0.2.40 Runtime Replay Direct Entry Scan

## Summary

Runtime replay no longer allocates a filtered replay-entry array before processing a batch.

## Changes

- Replaced replay pre-filtering with direct iteration over `replay.entries`.
- Skipped stale replay entries inline by comparing each sequence to the current cursor.
- Added source-contract coverage that rejects the older `replay.entries.filter(...)` path.

## Impact

- Removes one replay-batch array allocation from the PTY catch-up path.
- Reduces replay repair overhead when a batch mixes stale and fresh entries.
- Keeps replay semantics unchanged while tightening the model hot path.
