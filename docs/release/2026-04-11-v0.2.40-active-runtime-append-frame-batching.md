# 2026-04-11 v0.2.40 Active Runtime Append Frame Batching

## Summary

Visible runtime tabs now batch incremental PTY append writes per frame before sending them to xterm.

## Changes

- Added queued append buffering for active runtime tabs inside `TerminalStage`.
- Added frame-scheduled append flush for incremental `runtimeTerminalContent` growth.
- Cleared queued append data when the terminal needs a full reset, a tab becomes inactive, or runtime mode changes.
- Added source-contract coverage for append batching and reset-path queue clearing.

## Impact

- Cuts the number of xterm writes during bursty PTY output.
- Improves visible-tab smoothness during shell startup, burst replay, and heavy command output.
- Keeps full-content recovery correct because reset and reactivation paths still force a full raw content sync.
