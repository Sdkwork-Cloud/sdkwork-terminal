# Step 质量审计清单与复盘模板

## 1. 质量清单

- 是否明确对齐架构文档编号
- 是否明确写入范围
- 是否明确设计与非目标
- 是否明确测试与验证命令
- 是否明确检查点
- 是否明确串行/并行边界
- 是否明确风险与回滚
- 是否给出完成定义与下一步准入
- 是否给出能力兑现与架构回写结论
- 是否给出并行车道收口说明

## 2. 评分建议

| 维度 | 分值 |
| --- | --- |
| 架构对齐 | 20 |
| 落地可执行性 | 20 |
| 测试完整性 | 20 |
| 风险与回滚 | 20 |
| 证据与验收 | 20 |

建议：`>= 90` 才进入下一 step。

## 3. 一票否决项

- 没有能力兑现结论
- 没有架构回写
- 没有测试结果摘要
- 没有回滚路径
- 并行改了共享契约但没有收口记录

出现任一项，本 step 直接判定为 `Fail`。

## 4. 复盘结论等级

- `Pass`：可进入下一 step
- `Conditional Pass`：仅允许进入修补或收口 step，不允许扩新范围
- `Fail`：必须回退本 step 继续修正

## 5. 复盘模板

- 本步目标
- 完成项
- 未完成项
- 检查点通过情况
- 测试结果摘要
- 能力兑现结论
- 架构回写结论
- 关键风险
- 并行收口说明
- 复盘等级：`Pass / Conditional Pass / Fail`
- 是否允许进入下一 step
- 需要回写的架构文档
